Everything you need to stack frames

From raw FITS to a finished stack — debayering, alignment, calibration, and integration in one pipeline.

🚀

GPU-Accelerated Core

Every compute-heavy stage runs on CUDA — VNG debayer, Moffat convolution, Lanczos warp, and kappa-sigma integration use double-buffered stream overlap for maximum GPU utilization.

⭐

Automatic Star Detection & Alignment

Moffat PSF convolution with adaptive sigma threshold detects stars per frame. Triangle-matching + DLT computes a full homography — no manual reference or pre-computed transforms required. Matching runs on CPU or GPU via --match-device.

🎨

Full Color Output

Bayer pattern auto-detection from FITS BAYERPAT keyword. VNG demosaic produces separate R, G, B planes; all warp and integration stages run per-channel; output FITS uses NAXIS=3.

🔧

Calibration Frame Pipeline

Dark, flat, bias, and darkflat master generation via winsorized mean, median, or kappa-sigma stacking. Applied before debayering. Dead-pixel guard and flat normalization built-in.

💾

Flexible Output Formats

FITS (FP32), TIFF (FP32/FP16/INT16/INT8 + none/zip/lzw/rle), PNG (8/16-bit). Integer stretch bounds auto-derived per image or set manually. Format detected from file extension.

🖥️

CPU Fallback & Apple Silicon

Full OpenMP-parallelized CPU path via --cpu. Metal backend scaffolded for Apple Silicon with phased kernel porting. CPU-only builds require no NVIDIA GPU or CUDA runtime.

Processing Pipeline

Six stages, two execution paths. Pass --cpu to run everything with OpenMP instead of CUDA.

# Stage GPU path (default) CPU path (--cpu)
1 Debayering (star detection) VNG demosaic → luminance CUDA kernel VNG demosaic → luminance OpenMP
2 Star Detection Moffat PSF conv + σ threshold CUDA Moffat PSF conv + σ threshold OpenMP
3 Triangle Matching + Alignment Triangle asterism matching + DLT CPU CUDA
--match-device selects CPU or GPU; default follows stacking device		 Triangle asterism matching + DLT CPU
4 Debayering (warp) VNG demosaic → lum or R/G/B CUDA VNG demosaic → lum or R/G/B OpenMP
5 Lanczos-3 Warp nppiRemap + coord-map kernel CUDA 6-tap backward-map warp OpenMP
6 Integration Mini-batch κ-σ clipping CUDA Full κ-σ clipping OpenMP
ℹ️
Single-pass loading — each frame file is opened exactly once. Star detection, RANSAC, and warping all complete before the next frame is loaded. No intermediate files.
⚠️
Mismatch handling — frames that fail alignment (too few stars or triangle-matching mismatch) are skipped gracefully. A final summary prints successful / skipped counts.

# GPU build (CUDA 12 toolkit required)
cmake -B build -DCMAKE_BUILD_TYPE=Release \
      -DCMAKE_CUDA_COMPILER=/usr/local/cuda/bin/nvcc
cmake --build build --parallel $(nproc)

# CPU-only build (no CUDA toolkit needed)
cmake -B build -DCMAKE_BUILD_TYPE=Release -DDSO_ENABLE_CUDA=OFF
cmake --build build --parallel $(nproc)

# Run
./build/dso_stacker -f frames.csv -o stacked.fits

Prerequisites: CUDA Toolkit 12.x · CFITSIO 4.6.3 · libtiff · libpng · CMake ≥ 3.18

# Install C library dependencies via vcpkg
vcpkg install cfitsio tiff libpng --triplet x64-windows

# Configure
cmake -B build -G "Visual Studio 17 2022" -A x64 `
      -DCMAKE_TOOLCHAIN_FILE="$env:VCPKG_ROOT/scripts/buildsystems/vcpkg.cmake" `
      -DCMAKE_CUDA_COMPILER="C:/Program Files/NVIDIA GPU Computing Toolkit/CUDA/v12.6/bin/nvcc.exe"

# Compile
cmake --build build --config Release --parallel

Prerequisites: Visual Studio 2022 · CUDA Toolkit 12.x · vcpkg

# Apple Silicon — Metal scaffold backend
cmake -B build -DCMAKE_BUILD_TYPE=Release \
      -DDSO_ENABLE_CUDA=OFF \
      -DDSO_ENABLE_METAL=ON
cmake --build build --parallel

# Select Metal backend at runtime
./build/dso_stacker -f frames.csv -o stacked.fits --backend metal
🔨
Metal backend is scaffolded. Phase-1 wires backend selection and falls back to the CPU pipeline while Metal kernels are ported incrementally.

CLI Usage

Point it at a 2-column CSV frame list and choose your options. Star detection and alignment run automatically.

Input CSV format

filepath, is_reference
/data/frame1.fits, 1
/data/frame2.fits, 0
/data/frame3.fits, 0

Example commands

GPU stack (default)

dso_stacker -f frames.csv -o stacked.fits

CPU-only (no GPU required)

dso_stacker -f frames.csv -o stacked.fits --cpu

Color OSC camera (RGGB sensor)

dso_stacker -f frames.csv -o stacked.fits --bayer rggb --kappa 2.5 --iterations 5

With calibration frames

dso_stacker -f frames.csv -o stacked.fits \
    --bias  bias_frames.txt \
    --dark  dark_frames.txt \
    --flat  flat_frames.txt \
    --save-master-frames ./masters

16-bit TIFF with ZIP compression

dso_stacker -f frames.csv -o stacked.tiff --bit-depth 16 --tiff-compression zip

Output Formats

Desktop GUI

A PySide6 desktop app wrapping the CLI. Drag-and-drop frame management, all stacking options, and YAML project save/load.

  • ✓
    Drag-and-drop frames
    Drop FITS files onto Light, Dark, Flat, Bias, or Darkflat tabs
  • ✓
    Async FITS metadata
    Filename, path, size, and W×H loaded in background — UI never blocks
  • ✓
    Conditional options
    Kappa/iterations hidden for mean integration; compression hidden for non-TIFF; stretch bounds hidden for FP output
  • ✓
    YAML project files
    Save and reload complete project state — all frame lists and every option
  • ✓
    Live log + Abort
    Subprocess stdout/stderr streamed to collapsible log panel; Abort terminates without zombie processes
# Install Python deps
pip install PySide6 pyyaml

# Launch
python src/GUI/main.py

Or download a pre-built GUI bundle from Releases — no Python required.
